# ระบบข้อสอบ (ปรนัย / จับคู่ / อัตนัย) — Backend + Database

Backend จริงสำหรับระบบข้อสอบ พร้อม **แยกหน้านักเรียนกับผู้ดูแลออกจากกันคนละ URL**
เพื่อลดความสับสน และมีระบบ **รายชื่อนักเรียน + กำหนดสิทธิ์สอบตามห้อง** ให้นักเรียน
ล็อกอินด้วยรหัสนักเรียนแล้วระบบดึงชื่อ-สกุล-ห้อง และวิชาที่ต้องสอบขึ้นมาอัตโนมัติ
หลังสอบเสร็จคะแนนจะถูกบันทึกพร้อมชื่ออาจารย์ประจำวิชา และ**ส่งออกเป็น Excel**ได้ทันที

## หน้าเว็บ 2 หน้า แยกกันชัดเจน
| หน้า | URL | ใช้โดย |
|---|---|---|
| หน้านักเรียน | `http://<โดเมนของคุณ>/` | นักเรียนกรอกรหัสเข้าสอบ |
| หน้าผู้ดูแล/อาจารย์ | `http://<โดเมนของคุณ>/admin` | จัดการชุดข้อสอบ, รายชื่อนักเรียน, ดูผลสอบ, ส่งออก Excel |

## โครงสร้างโปรเจกต์
```
exam-backend/
├── server.js              ← เซิร์ฟเวอร์ Express + REST API
├── package.json
├── .env.example            ← คัดลอกเป็น .env แล้วตั้งรหัสผู้ดูแล
├── data/
│   └── db.json              ← ไฟล์ฐานข้อมูล (สร้างอัตโนมัติตอนรันครั้งแรก)
└── public/
    ├── student.html          ← หน้านักเรียน (ล็อกอินด้วยรหัสนักเรียน)
    └── admin.html            ← หน้าผู้ดูแล/อาจารย์ (ต้องกรอก Admin Key)
```

## วิธีติดตั้งและรัน
ต้องมี [Node.js](https://nodejs.org) เวอร์ชัน 18 ขึ้นไป

```bash
cd exam-backend
npm install                 # express, dotenv, xlsx — ไม่มี native module ติดตั้งง่ายทุกเครื่อง
cp .env.example .env        # แก้ ADMIN_KEY ในไฟล์ .env เป็นรหัสของจริง
npm start
```
- นักเรียนเข้า `http://localhost:3000`
- ผู้ดูแล/อาจารย์เข้า `http://localhost:3000/admin`

## ขั้นตอนการใช้งานของผู้ดูแล
1. เปิด `/admin` แล้วกรอก Admin Key เข้าสู่ระบบ
2. แท็บ **"รายชื่อนักเรียน"** — วางรายชื่อจาก Excel (รหัส, ชื่อ, นามสกุล, ห้อง คนละบรรทัด)
   คัดลอกมาจาก Excel ได้เลย ระบบรองรับทั้ง tab และ comma เป็นตัวคั่น
3. แท็บ **"ชุดข้อสอบ"** — สร้าง/แก้ไขชุดข้อสอบ พร้อม:
   - กำหนด **ห้องที่มีสิทธิสอบ** (ถ้าไม่กำหนด = เปิดให้ทุกห้องสอบได้)
   - ระบุ **อาจารย์ประจำวิชา** (ชื่อ + อีเมล) เพื่อผูกกับผลสอบที่จะถูกส่งเข้ามา
4. นักเรียนเข้า `/` กรอกรหัสนักเรียน → ระบบดึงชื่อ-สกุล-ห้องอัตโนมัติ → เห็นเฉพาะวิชาที่ห้องตนเองมีสิทธิสอบ
5. หลังนักเรียนสอบเสร็จ ผลจะเข้าแท็บ **"ผลสอบนักเรียน"** ทันที กรองตามรายวิชาได้ แล้วกด
   **"ส่งออก Excel"** เพื่อได้ไฟล์ .xlsx (รหัส, ชื่อ, ห้อง, รายวิชา, อาจารย์, คะแนนแต่ละส่วน, คะแนนรวม)
   พร้อมส่งต่อให้อาจารย์ประจำวิชาทางอีเมลหรือช่องทางอื่นได้เลย

## REST API ที่มีให้

| Method | Endpoint | สิทธิ์ | คำอธิบาย |
|---|---|---|---|
| GET | `/api/students/:id` | สาธารณะ | นักเรียนล็อกอินด้วยรหัส → คืนชื่อ/นามสกุล/ห้อง |
| GET | `/api/students` | ผู้ดูแล | รายชื่อนักเรียนทั้งหมด (`?classRoom=` กรองได้) |
| POST | `/api/students` | ผู้ดูแล | เพิ่มนักเรียน 1 คน |
| POST | `/api/students/bulk` | ผู้ดูแล | นำเข้ารายชื่อจำนวนมาก (วางข้อความหลายบรรทัด) |
| PUT | `/api/students/:id` | ผู้ดูแล | แก้ไขข้อมูลนักเรียน |
| DELETE | `/api/students/:id` | ผู้ดูแล | ลบนักเรียน |
| GET | `/api/classes` | ผู้ดูแล | รายชื่อห้องทั้งหมดที่มีในระบบ |
| GET | `/api/sets` | สาธารณะ | รายการชุดข้อสอบ (`?classRoom=` กรองสิทธิ์ตามห้อง) |
| GET | `/api/sets/:key` | สาธารณะ | ชุดข้อสอบเดียว |
| POST | `/api/sets` | ผู้ดูแล | สร้างชุดข้อสอบ (รวม assignedClasses, subjectTeacherName) |
| PUT | `/api/sets/:key` | ผู้ดูแล | แก้ไขชุดข้อสอบ |
| POST | `/api/sets/:key/duplicate` | ผู้ดูแล | ทำสำเนาชุดข้อสอบ |
| DELETE | `/api/sets/:key` | ผู้ดูแล | ลบชุดข้อสอบ |
| POST | `/api/results` | สาธารณะ | นักเรียนส่งผลสอบเข้าระบบ |
| GET | `/api/results` | ผู้ดูแล | ดูผลสอบทั้งหมด (`?setKey=` กรองรายวิชา) |
| DELETE | `/api/results/:id` | ผู้ดูแล | ลบผลสอบ 1 รายการ |
| GET | `/api/export/results.xlsx` | ผู้ดูแล | ดาวน์โหลด Excel (`?setKey=` กรองรายวิชา) |
| POST | `/api/admin/verify` | - | ตรวจสอบรหัสผู้ดูแล (ใช้ตอน login) |

เส้นทางที่ต้อง "สิทธิ์ผู้ดูแล" ต้องแนบ header `x-admin-key: <รหัสจาก .env>` มาด้วยทุกครั้ง
(หน้า `/admin` จัดการเรื่องนี้ให้อัตโนมัติหลัง login สำเร็จ)

## กติกาสิทธิ์การสอบ (assignedClasses)
- ชุดข้อสอบที่ **ไม่ได้กำหนดห้อง** (assignedClasses ว่าง) = เปิดให้ทุกห้องเห็นและสอบได้ (ค่าเริ่มต้น
  เพื่อความเข้ากันได้กับชุดข้อสอบเก่า)
- ชุดข้อสอบที่ **กำหนดห้องไว้** = เฉพาะนักเรียนที่ห้องตรงกับที่กำหนดเท่านั้นที่จะเห็นวิชานี้ในหน้า
  เลือกวิชาของตนเอง

## ฐานข้อมูลทำงานอย่างไร
ใช้ไฟล์ `data/db.json` เก็บ 3 ส่วน: `sets` (ชุดข้อสอบ), `students` (รายชื่อนักเรียน),
`results` (ผลสอบ) เลือกวิธีนี้แทน SQL database เพราะไม่ต้องพึ่ง native module เลย ติดตั้งง่าย
ทุกเครื่อง ถ้าต้องขยายเป็น PostgreSQL/MySQL ในอนาคต ให้แก้แค่ฟังก์ชัน `readDB()`/`writeDB()`
ใน `server.js` เท่านั้น จุดอื่นไม่ต้องแก้ **สำรองข้อมูล:** คัดลอกไฟล์ `data/db.json` เก็บไว้เป็นระยะ

## Deploy ให้ใช้งานจริง
1. **Railway / Render / Fly.io** — เชื่อม GitHub repo ตั้งค่า Environment Variable `ADMIN_KEY`
   แล้ว deploy (ฟรี tier พอสำหรับใช้ในโรงเรียน)
2. **VPS ของโรงเรียนเอง** — ติดตั้ง Node.js, รันด้วย `pm2 start server.js`, ตั้ง nginx reverse
   proxy + HTTPS (Let's Encrypt)
3. **สำคัญ:** เปลี่ยน `ADMIN_KEY` เป็นรหัสที่คาดเดายาก และเปิด HTTPS เสมอเมื่อใช้งานจริง

## หมายเหตุด้านความปลอดภัย
- ระบบยืนยันตัวตนผู้ดูแลเป็นรหัสเดียว (shared key) เหมาะกับผู้ดูแล/อาจารย์ไม่กี่คน ถ้าต้องแยก
  สิทธิ์ต่ออาจารย์แต่ละคนจริงจัง ควรทำระบบ user/password ต่อบัญชีแทน
- รหัสนักเรียนใช้แทน "รหัสผ่าน" เข้าสอบ ควรแจ้งนักเรียนไม่ให้เผยแพร่รหัสตนเอง และอาจพิจารณา
  เพิ่มการยืนยันตัวตนอีกชั้น (เช่น วันเกิด) หากต้องการความปลอดภัยสูงขึ้น
- ควรเพิ่ม rate limiting กับ endpoint `/api/results`, `/api/students/:id` และ `/api/admin/verify`
- ควรเปิด HTTPS เสมอเมื่อใช้งานจริงนอกเครื่อง localhost
