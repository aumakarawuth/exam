# ระบบข้อสอบ (ปรนัย / จับคู่ / อัตนัย) — Backend + Database

Backend จริงสำหรับระบบข้อสอบ **แยกหน้านักเรียนกับผู้ดูแลออกจากกันคนละ URL**, มีระบบ
**รายชื่อนักเรียน + กำหนดสิทธิ์สอบตามห้อง**, แบ่งข้อสอบเป็น **กลางภาค/ปลายภาค**,
**คะแนนเต็มรวม 20 คะแนนต่อวิชา**, **ตรวจข้อสอบฝั่งเซิร์ฟเวอร์เท่านั้น** (เฉลยไม่ถูกส่งไปที่
เบราว์เซอร์นักเรียนเด็ดขาด) และรองรับ **2 โหมดประกาศผล**: ให้อาจารย์ตรวจสอบก่อนจึงประกาศ
หรือประกาศคะแนนอัตโนมัติทันทีที่นักเรียนส่งคำตอบ — นักเรียนตรวจสอบผลคะแนนของตัวเองได้ที่หน้า
"ตรวจสอบผลคะแนน" (แสดงคะแนนเฉพาะวิชาที่ประกาศผลแล้วเท่านั้น) มีการตรวจจับคลิกขวา/พยายาม
คัดลอกข้อความระหว่างสอบ, รองรับ**สุ่มลำดับโจทย์และตัวเลือก**ต่อคน, ข้อสอบปรนัยแสดงทีละข้อ
พร้อมปุ่มไปข้อถัดไป/ก่อนหน้าและตารางแสดงสถานะการตอบแบบปฏิทิน, **บังคับตอบให้ครบทุกข้อ**ก่อน
ส่งคำตอบโดยสมัครใจ (แต่ไม่บล็อกการส่งอัตโนมัติเมื่อหมดเวลา เพื่อไม่ให้คำตอบที่ทำไว้สูญหาย), และมี
**Dark Mode** ให้สลับใช้งานได้ ไม่มีการสร้าง PDF ให้นักเรียนดาวน์โหลดอีกต่อไป

## หน้าเว็บ 3 หน้า แยกกันชัดเจน
| หน้า | URL | ใช้โดย |
|---|---|---|
| หน้านักเรียน | `http://<โดเมนของคุณ>/` | นักเรียนกรอกรหัสเข้าสอบ / ตรวจสอบผลคะแนน |
| หน้าผู้ดูแล | `http://<โดเมนของคุณ>/admin` | จัดการชุดข้อสอบทั้งหมด, รายชื่อนักเรียน, บัญชีอาจารย์, ผลสอบทุกวิชา (เข้าด้วย Admin Key) |
| หน้าอาจารย์ผู้สอน | `http://<โดเมนของคุณ>/teacher` | อาจารย์แต่ละคนจัดการเฉพาะข้อสอบและผลสอบของวิชาตัวเอง (เข้าด้วย username/password) |

## โครงสร้างโปรเจกต์
```
exam-backend/
├── server.js              ← เซิร์ฟเวอร์ Express + REST API + ตรวจข้อสอบ (grading)
├── package.json
├── .env.example            ← คัดลอกเป็น .env แล้วตั้งรหัสผู้ดูแล
├── data/
│   └── db.json              ← ไฟล์ฐานข้อมูล (สร้างอัตโนมัติตอนรันครั้งแรก)
└── public/
    ├── student.html          ← หน้านักเรียน (ล็อกอินด้วยรหัสนักเรียน)
    ├── admin.html            ← หน้าผู้ดูแล (ต้องกรอก Admin Key) จัดการทุกวิชา/บัญชีอาจารย์/รายชื่อนักเรียน
    └── teacher.html          ← หน้าอาจารย์ผู้สอน (username/password) เห็นเฉพาะวิชาของตัวเอง
```

## วิธีติดตั้งและรัน
ต้องมี [Node.js](https://nodejs.org) เวอร์ชัน 18 ขึ้นไป

```bash
cd exam-backend
npm install                 # express, dotenv, xlsx — ไม่มี native module ติดตั้งง่ายทุกเครื่อง
cp .env.example .env        # แก้ ADMIN_KEY ในไฟล์ .env เป็นรหัสของจริง
npm start
```
- นักเรียนเข้า `http://localhost:3000`
- ผู้ดูแล/อาจารย์เข้า `http://localhost:3000/admin`

## ขั้นตอนการใช้งานของผู้ดูแล
1. เปิด `/admin` แล้วกรอก Admin Key เข้าสู่ระบบ
2. แท็บ **"รายชื่อนักเรียน"** — วางรายชื่อจาก Excel (รหัส, ชื่อ, นามสกุล, ห้อง คนละบรรทัด)
3. แท็บ **"ชุดข้อสอบ"** — สร้าง/แก้ไขชุดข้อสอบ พร้อม:
   - เลือก **ประเภทข้อสอบ**: กลางภาค หรือ ปลายภาค
   - กำหนด **ห้องที่มีสิทธิสอบ** (ไม่กำหนด = เปิดให้ทุกห้องสอบได้)
   - ระบุ **อาจารย์ประจำวิชา** (ชื่อ + อีเมล)
   - ตั้งค่า **โหมดประกาศผล**: "ต้องตรวจสอบก่อนจึงประกาศ" (ค่าเริ่มต้น) หรือ "ประกาศคะแนนให้
     นักเรียนทันทีที่ส่งคำตอบ" (เหมาะกับข้อสอบที่ตรวจอัตโนมัติแม่นยำ เช่น ไม่มีอัตนัย)
   - เปิด/ปิด **สุ่มลำดับโจทย์** และ **สุ่มลำดับตัวเลือกปรนัย** ได้อิสระต่อชุดข้อสอบ
   - ตั้งคะแนนแต่ละข้อ — มีตัวบอกคะแนนรวมแบบเรียลไทม์ เตือนถ้ายังไม่เท่ากับ **20 คะแนนเต็ม**
4. นักเรียนเข้า `/` กรอกรหัสนักเรียน → เห็นเฉพาะวิชาที่ห้องตนเองมีสิทธิสอบ → ทำข้อสอบ (ปรนัยแสดง
   ทีละข้อ มีปุ่มถัดไป/ก่อนหน้าและตารางเลขข้อบอกสถานะตอบแล้ว/ยังไม่ตอบ) → ต้องตอบครบทุกข้อ
   ก่อนกด "จบการสอบและส่งคำตอบ" ได้ (ถ้าขาดข้อไหน ระบบพาไปยังข้อนั้นให้อัตโนมัติ) → กดส่งคำตอบ
   → ระบบขึ้น **"คุณได้ส่งคำตอบเรียบร้อยแล้ว"** เท่านั้น **ไม่แสดงคะแนนทันที**
5. คะแนนถูกตรวจโดยเซิร์ฟเวอร์ทันทีที่ส่ง:
   - ถ้าชุดข้อสอบตั้งเป็น **"ประกาศอัตโนมัติ"** → นักเรียนเห็นคะแนนได้ทันทีที่หน้า "ตรวจสอบผล
     คะแนน" โดยไม่ต้องรอผู้ดูแล
   - ถ้าเป็นค่าเริ่มต้น **"ต้องตรวจสอบก่อน"** → ต้องเข้าแท็บ **"ผลสอบนักเรียน"** กด **"ดูราย
     ละเอียด"** เพื่อตรวจคำตอบเทียบเฉลย แล้วกด **"ยังไม่ประกาศ 🔒 → ประกาศแล้ว ✅"** เพื่อเปิดให้
     นักเรียนเห็นคะแนนของตน
6. กรองผลตามประเภทข้อสอบ/รายวิชา แล้วกด **"ส่งออก Excel"** เพื่อได้ไฟล์ .xlsx ส่งต่อให้
   อาจารย์ประจำวิชา

## ความปลอดภัยของการตรวจข้อสอบ
- **เฉลยไม่เคยถูกส่งไปที่เบราว์เซอร์นักเรียน** — endpoint สาธารณะ `/api/sets` ตัดฟิลด์เฉลย
  ออกหมด (`answer`, `correctMap`, `keywords`) เหลือแค่โจทย์/ตัวเลือกที่ต้องแสดงผล
- นักเรียนส่ง**คำตอบดิบ**เท่านั้น (`POST /api/results`) เซิร์ฟเวอร์เป็นผู้ตรวจให้คะแนนทั้งหมด
  และ**ไม่ส่งคะแนนกลับไปที่หน้าจอนักเรียนตอนส่งคำตอบเลย** ต้องเข้าหน้า "ตรวจสอบผลคะแนน" อีกครั้ง
  ซึ่งจะซ่อนคะแนนไว้จนกว่าฟิลด์ `published` จะเป็น true เท่านั้น
- **การสุ่มลำดับโจทย์/ตัวเลือกทำที่ฝั่งไคลเอนต์เท่านั้น ไม่กระทบการตรวจ** — ระบบตรวจโดยอ้างอิง
  `id` ของแต่ละข้อและ**ดัชนีเดิม**ของตัวเลือก ไม่ใช่ลำดับที่แสดงผล จึงสุ่มได้อย่างปลอดภัย
- **การบังคับตอบให้ครบ** มีทั้งฝั่งไคลเอนต์ (พาไปยังข้อที่ขาด) และฝั่งเซิร์ฟเวอร์ (ปฏิเสธคำขอที่
  ไม่ครบ) แต่จะ**ไม่บล็อกการส่งอัตโนมัติเมื่อหมดเวลา** เพื่อไม่ให้คำตอบที่ทำไว้บางส่วนสูญหาย
- เฉพาะ `GET /api/admin/sets` (ต้องมี Admin Key) เท่านั้นที่คืนข้อมูลเฉลยเต็มรูปแบบ ใช้สำหรับ
  หน้าแก้ไขข้อสอบและหน้าดูรายละเอียดคำตอบของผู้ดูแลเท่านั้น

## การตรวจจับพฤติกรรมระหว่างสอบ
บันทึกไว้ในผลสอบทุกครั้ง (ดูได้ในแท็บผลสอบและไฟล์ Excel):
- **สลับแท็บ/สลับหน้าต่าง** (`tabSwitches`)
- **คลิกขวา** (`rightClickAttempts`) — ระบบบล็อกเมนูคลิกขวาระหว่างทำข้อสอบ
- **พยายามคัดลอก/ตัดข้อความ** (`copyAttempts`) — ระบบบล็อกการคัดลอกระหว่างทำข้อสอบ
- **โหลดหน้าใหม่** (`reloadCount`)

## กติกาสิทธิ์การสอบ (assignedClasses)
ชุดข้อสอบที่ไม่ได้กำหนดห้อง = เปิดให้ทุกห้องเห็นและสอบได้ (ค่าเริ่มต้น) ถ้ากำหนดห้องไว้
เฉพาะนักเรียนห้องที่ตรงกันเท่านั้นที่จะเห็นวิชานี้ในหน้าเลือกวิชาของตนเอง

## REST API ที่มีให้

| Method | Endpoint | สิทธิ์ | คำอธิบาย |
|---|---|---|---|
| GET | `/api/students/:id` | สาธารณะ | นักเรียนล็อกอินด้วยรหัส → คืนชื่อ/นามสกุล/ห้อง |
| GET | `/api/students/:id/results` | สาธารณะ | นักเรียนตรวจสอบผลคะแนนของตนเอง (ซ่อนคะแนนถ้ายังไม่ประกาศ) |
| GET | `/api/students` | ผู้ดูแล | รายชื่อนักเรียนทั้งหมด (`?classRoom=` กรองได้) |
| POST | `/api/students` | ผู้ดูแล | เพิ่มนักเรียน 1 คน |
| POST | `/api/students/bulk` | ผู้ดูแล | นำเข้ารายชื่อจำนวนมาก |
| PUT/DELETE | `/api/students/:id` | ผู้ดูแล | แก้ไข/ลบนักเรียน |
| GET | `/api/classes` | ผู้ดูแล | รายชื่อห้องทั้งหมดที่มีในระบบ |
| GET | `/api/exam-types` | สาธารณะ | รายการประเภทข้อสอบ (กลางภาค/ปลายภาค) |
| GET | `/api/sets` | สาธารณะ | ชุดข้อสอบ **ไม่มีเฉลย** (`?classRoom=` กรองสิทธิ์) |
| GET | `/api/admin/sets` | ผู้ดูแล | ชุดข้อสอบ **พร้อมเฉลยเต็ม** (ใช้แก้ไข/ตรวจทาน) |
| POST/PUT/DELETE | `/api/sets...` | ผู้ดูแล | สร้าง/แก้ไข/ลบชุดข้อสอบ (รวม shuffle/publishMode) |
| POST | `/api/results` | สาธารณะ | นักเรียนส่ง**คำตอบดิบ** เซิร์ฟเวอร์ตรวจให้ ไม่คืนคะแนนกลับ |
| GET | `/api/results` | ผู้ดูแล | ดูผลสอบ+คะแนน (`?setKey=`, `?examType=` กรองได้) |
| PATCH | `/api/results/:id` | ผู้ดูแล | สลับสถานะ "ประกาศผลแล้ว" ของผลสอบ 1 รายการ |
| POST | `/api/sets/:key/publish` | ผู้ดูแล | ประกาศผลทุกคนในวิชานั้นพร้อมกัน |
| DELETE | `/api/results/:id` | ผู้ดูแล | ลบผลสอบ 1 รายการ |
| GET | `/api/export/results.xlsx` | ผู้ดูแล | ดาวน์โหลด Excel (`?setKey=`, `?examType=` กรองได้) |
| POST | `/api/admin/verify` | - | ตรวจสอบรหัสผู้ดูแล |
| GET/POST | `/api/teachers` | ผู้ดูแล | รายชื่อ/เพิ่มบัญชีอาจารย์ |
| DELETE | `/api/teachers/:id` | ผู้ดูแล | ลบบัญชีอาจารย์ (ข้อสอบเดิมยังอยู่แต่ไม่ผูกบัญชีแล้ว) |
| POST | `/api/teacher/login` | สาธารณะ | อาจารย์ล็อกอินด้วย username/password → ได้ session token |
| GET/POST/PUT/DELETE | `/api/teacher/sets...` | อาจารย์ (token) | จัดการเฉพาะชุดข้อสอบของตัวเอง |
| GET/PATCH | `/api/teacher/results...` | อาจารย์ (token) | ดู/ประกาศผลเฉพาะวิชาของตัวเอง |
| GET | `/api/teacher/export/results.xlsx` | อาจารย์ (token) | ส่งออก Excel เฉพาะวิชาของตัวเอง |

## ฐานข้อมูลทำงานอย่างไร
ใช้ไฟล์ `data/db.json` เก็บ 3 ส่วน: `sets`, `students`, `results` แทน SQL database เพื่อไม่ต้อง
พึ่ง native module เลย ถ้าต้องขยายเป็น PostgreSQL/MySQL ในอนาคต ให้แก้แค่ `readDB()`/`writeDB()`
ใน `server.js` **สำรองข้อมูล:** คัดลอกไฟล์ `data/db.json` เก็บไว้เป็นระยะ

## อัปเดตล่าสุด
- **ส่วนที่ไม่มีเนื้อหาจะไม่แสดงให้นักเรียนเห็นเลย** — ถ้าไม่ได้เพิ่มส่วนจับคู่/อัตนัย นักเรียนจะเห็นแค่ส่วนปรนัยเท่านั้น
- **หน้าสร้างข้อสอบใหม่ในฝั่งผู้ดูแล** ออกแบบใหม่ทั้งหมด: เพิ่ม/ลบส่วนจับคู่-อัตนัยได้เองผ่านเมนู "+ เพิ่มส่วน",
  และแต่ละส่วนแสดงเป็นรายการสรุปทีละข้อ (เหมือนหน้าที่นักเรียนเห็น) พร้อมปุ่ม "+ เพิ่มข้อ" ที่เปิดฟอร์มกรอกทีละข้อ
  แทนฟอร์มยาวที่สับสน
- **ไม่ต้องตั้งคะแนนรายข้อเองอีกต่อไป** — พิมพ์แค่ "คะแนนรวมส่วนนี้" ที่หัวข้อแต่ละส่วน ระบบหารเฉลี่ยให้อัตโนมัติ
  ทุกข้อในส่วนนั้น (ปรนัยไม่ตั้งอะไรเลยจะได้ค่าเริ่มต้นข้อละ 1 คะแนน)
- **กำหนดเวลาลงทะเบียนสอบ + รหัสเข้าสอบย้อนหลัง** — ตั้งเวลาสิ้นสุดการสอบตามปกติ และรหัสพิเศษสำหรับนักเรียนที่มาสาย
  ได้ในหน้าสร้าง/แก้ไขชุดข้อสอบ ระบบตรวจสอบทั้งฝั่งหน้าเว็บและฝั่งเซิร์ฟเวอร์
- **1 รหัสนักเรียน ทำข้อสอบแต่ละวิชาได้ครั้งเดียว** บังคับที่เซิร์ฟเวอร์ ป้องกันการส่งซ้ำแม้จะพยายามเลี่ยงหน้าเว็บ
- **ล้างค่าตรวจจับพฤติกรรม** (สลับแท็บ/คลิกขวา/คัดลอก) ทันทีที่ส่งคำตอบเสร็จ ไม่ตกค้างข้ามไปวิชาอื่นหรือเครื่องที่ใช้ร่วมกัน

## อัปเดตล่าสุด (รอบ 3)
- **หน้าสร้าง/แก้ไขชุดข้อสอบเปลี่ยนเป็น Wizard เต็มหน้าจอ** เดินตามขั้นตอนทีละหน้า (ข้อมูล →
  สิทธิ์/อาจารย์ → เลือกส่วน → รายการส่วนข้อสอบ) คล้ายกับหน้าที่นักเรียนใช้ทำข้อสอบจริง — เข้าไป
  แก้แต่ละส่วนทีละหน้า ไม่ปนกันเป็นฟอร์มยาวอีกต่อไป ตั้งค่าประกาศผล/สุ่มข้อสอบอยู่ที่หัวข้อ
  ส่วนปรนัยโดยเฉพาะ
- **ระบบบัญชีอาจารย์ผู้สอน** — ผู้ดูแล (Admin Key) เพิ่มบัญชีอาจารย์ได้ในแท็บ "อาจารย์ผู้สอน"
  (ชื่อ, นามสกุล, username, password) อาจารย์เข้าสู่ระบบที่หน้า **`/teacher`** ด้วยบัญชีของตัวเอง
  จะเห็น**เฉพาะ**ชุดข้อสอบและผลสอบของวิชาที่ตัวเองสร้างเท่านั้น สร้าง/แก้ไข/ลบข้อสอบและประกาศผล
  ของตัวเองได้เต็มรูปแบบ โดยไม่เห็นข้อมูลของอาจารย์ท่านอื่นหรือรายชื่อนักเรียนทั้งหมด
- **จัดกลุ่มชุดข้อสอบตามชื่อวิชา** — ชุดข้อสอบกลางภาคและปลายภาคของวิชาเดียวกันจะถูกจัดกลุ่มแสดง
  รวมกันในหน้ารายการ (ทั้งฝั่งผู้ดูแลและฝั่งอาจารย์) พับ/ขยายดูได้ ไม่ต้องไล่หาทีละชุดเมื่อมีข้อสอบ
  จำนวนมาก

## Deploy ให้ใช้งานจริง
ดูรายละเอียดใน `DEPLOY.md` (Railway, Render, หรือ VPS โรงเรียนเองพร้อม PM2 + Nginx + HTTPS)


## หมายเหตุด้านความปลอดภัย
- ระบบยืนยันตัวตนผู้ดูแลเป็นรหัสเดียว (shared key) เหมาะกับผู้ดูแล/อาจารย์ไม่กี่คน
- รหัสนักเรียนใช้แทน "รหัสผ่าน" เข้าสอบ ควรแจ้งนักเรียนไม่ให้เผยแพร่รหัสตนเอง
- ควรเพิ่ม rate limiting กับ `/api/results`, `/api/students/:id`, `/api/admin/verify`
- ควรเปิด HTTPS เสมอเมื่อใช้งานจริงนอกเครื่อง localhost
